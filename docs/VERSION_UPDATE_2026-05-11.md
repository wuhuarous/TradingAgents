# 版本更新说明 - 2026-05-11

## 更新目标

本次版本把系统从“页面展示型原型”继续推进到“可用的模拟量化工作台”。重点解决四类问题：

- 行情总览数据像固定样例，缺少真实市场温度和板块线索。
- 智能选股和模拟交易需要从全市场预筛，不能依赖少量写死股票池。
- 新闻资讯需要定时更新、标准化入库，并能用于后续复盘。
- 仪表盘实时资产、持仓盈亏和模拟账户状态需要更稳定。

## 主要更新

### 1. 行情总览升级

- A 股涨幅榜、跌幅榜改为基于全量实时快照生成，不再只从固定热门股票池中排序。
- 新增市场温度：
  - 覆盖股票数
  - 上涨家数
  - 下跌家数
  - 涨停家数
  - 跌停家数
- 新增热门板块：
  - 同花顺行业板块
  - 同花顺概念资金流
  - 板块涨跌幅、净流入、领涨股、领涨股涨幅
- 新增成交额榜，用于观察资金真实活跃方向。
- 热门标的改为按成交额动态生成，减少“固定样例数据”的误导。

### 2. 新闻资讯刷新机制

- 新闻接口改为一小时刷新模型：
  - 最近 1 小时内有标准化新闻时优先读库。
  - 超过 1 小时自动重新聚合资讯。
  - 页面提供手动“刷新资讯”入口。
- 新增定时任务 `news_refresh`：
  - 每天 08:00-23:00 整点刷新。
  - 同步 A 股、港股、美股资讯。
  - 结果写入标准化新闻表，用于后续情绪因子和复盘。
- 过滤明显过期的宏观日历项，避免旧资讯污染当前新闻流。

### 3. 全市场候选池

- 新增 ClickHouse `candidate_pool` 表。
- 新增轻量候选池刷新流程：
  - 使用全市场实时快照和历史日线特征做预筛。
  - 先筛出候选池，再做较重的策略评分。
  - 避免模拟交易接口直接全市场深度扫描导致超时。
- 调度器新增候选池刷新任务：
  - 09:25 盘前候选池
  - 10:35 接近 10:40 买点前刷新
  - 13:55 接近 14:00 买点前刷新

### 4. 模拟交易与短线策略稳定性

- 模拟交易候选评分支持候选池优先，候选池为空时自动刷新或降级。
- 候选评分支持并行执行和时间预算，降低接口超时概率。
- 修复模拟交易运行结果中 numpy/pandas 类型导致 JSON 序列化失败的问题。
- 因子、行情、新闻等输入数据继续持久化，为后续复盘回填提供样本。
- 保留 A 股交易约束：
  - T+1
  - 涨跌停不可买卖
  - 手续费、过户费、印花税
  - 成交量参与上限
  - 停牌/无成交保护

### 5. 仪表盘实时资产修复

- 修复“可用现金被误认为总资产”的显示误解。
- 仪表盘总资产改为前端使用最新持仓聚合校验：
  - 总资产 = 可用现金 + 最新持仓市值
  - 累计盈亏 = 总资产 - 初始资金
- 持仓价格刷新改为小批量实时拉取，不再为了少量持仓触发全市场扫描。
- 仪表盘账户概览每 15 秒刷新一次。

### 6. Windows 时区兼容

- 新增 `tradingAgents/utils/timezone.py`。
- Windows 缺少 `tzdata` 时，`Asia/Shanghai` 自动降级为 UTC+8 固定时区。
- 避免后端在 Windows 环境因为 `ZoneInfo("Asia/Shanghai")` 启动失败。

## 已知限制与下一步

- A 股全市场行情依赖免费数据源，实时性和可用性仍受上游影响。
- 港股、美股全市场榜单目前仍受免费数据源限制，暂以配置池和外部新闻聚合作为补充。
- 热门板块接口来自 AkShare/同花顺链路，偶发代理或上游失败时会降级为空。
- 涨幅榜中首日新股涨幅会非常夸张，下一步应增加新股/次新股标记或过滤。
- 智能选股页还需要进一步统一短线 100 分模型、优质股模型和条件筛选逻辑。
- 后续建议增加：
  - 板块强度历史趋势
  - 龙虎榜数据
  - 分钟级买点验证
  - 策略排行榜按样本外表现排序
  - 模拟交易观察模式和自动买入开关更细粒度控制
