# AI 量化交易优化路线

## 目标

本轮优化把“追求 50% 年化收益”拆成可执行、可观测、可复盘的工程问题。系统不承诺固定收益，而是用数据质量、策略验证、风控执行和复盘学习来提高模拟交易的可解释性与可迭代性。

## 已落地

1. 新增后端诊断接口 `/api/simulation/readiness`，输出 AI 量化准备度。
2. 诊断维度覆盖数据可信度、策略验证、风控约束、交易执行、复盘学习、AI 稳定性。
3. 仪表板新增“AI 量化准备度”模块，可以直接看到当前短板。
4. 模拟训练页新增“交易前置条件”模块，展示训练轮次、阻塞项和下一步动作。

## 为什么先做准备度

高收益目标不能只靠 LLM 选股。真正影响系统上限的是：

- 行情、财务、新闻和舆情是否稳定、可追溯、可降级。
- 策略信号是否能在模拟盘和历史样本中反复验证。
- 买卖点是否受仓位、止损、止盈和最大回撤约束。
- 每次交易是否有完整复盘，能回填 1/5/20 日收益。
- AI 是否有规则降级能力，避免模型余额、限流或幻觉影响交易。

## 后续建议

1. 接入更完整的历史回测，至少输出年化收益、最大回撤、夏普比率、胜率和盈亏比。
2. 将复盘日志升级为结构化样本库，记录买入后 1/5/20 日收益和新闻变化。
3. 把行情源切换做成真正的 provider 路由，引入熔断、备用源和数据新鲜度评分。
4. 深度分析改成异步任务，前端通过 WebSocket 或 SSE 展示执行进度。
5. 实盘前增加券商适配层、订单幂等、成交回报、滑点模型和全局熔断开关。
