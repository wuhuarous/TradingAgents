# TradingAgents 第二版需求文档

## 1. 版本目标

第二版目标是把当前系统从“可演示的 AI 股票分析与模拟交易工具”升级为“可验证、可复盘、可训练、可控风险的量化研究与模拟交易系统”。

本版本不承诺固定收益率，而是围绕用户提出的年化 50% 目标，建立一套可以持续验证策略有效性、记录失败原因、校准因子权重的闭环。

核心目标：

- 全市场股票选择不再依赖硬编码股票池。
- 行情、新闻、舆情、财报、训练样本、交易记录、复盘结果全部持久化。
- 选股结果能解释：为什么进入 Top10，为什么买入、观望或规避。
- 模拟交易状态在服务重启后可恢复。
- 可以通过回测和复盘验证策略是否接近收益目标。
- 前端能直观看到收益、风险、数据质量、策略依据和复盘结果。

## 2. 用户核心场景

### 2.1 全市场智能选股

用户进入“智能选股”页面，选择市场、策略目标、风险偏好后，系统从全市场股票宇宙中筛选：

- Top10 优质股
- Top10 潜力股
- 今日买入候选
- 观察名单
- 规避名单

每只股票必须展示：

- 综合分
- 质量、成长、估值、趋势、情绪、风险因子拆解
- 入选原因
- 风险提示
- 数据来源与更新时间

### 2.2 自动模拟交易

用户开启模拟交易后，系统按策略周期自动运行：

- 读取股票宇宙
- 拉取行情、财报、新闻、舆情
- 计算因子得分
- 生成买入/卖出/持有动作
- 记录订单、持仓、现金、净值
- 服务重启后继续恢复账户状态

### 2.3 复盘训练

系统对每次候选和交易进行后验复盘：

- T+1、T+5、T+20、T+60 收益
- 是否触发止损/止盈
- 当时情绪是否误判
- 因子贡献是否有效
- 策略权重是否需要调整

复盘结果用于后续策略参数优化。

### 2.4 回测验证

用户选择时间区间、市场、策略参数，系统运行历史回测：

- 输出年化收益
- 最大回撤
- 胜率
- 盈亏比
- 夏普比率
- 换手率
- 每笔交易明细
- 净值曲线

## 3. 功能模块

## 3.1 股票宇宙管理

目标：统一管理全市场候选范围，替代散落代码中的股票池。

功能：

- 支持 A 股、港股、美股股票宇宙。
- 支持从数据源同步全市场列表。
- 支持配置种子股票、黑名单、白名单。
- 自动过滤 ST、退市、停牌、低成交额、低流动性标的。
- 支持行业、市值、成交额、上市年限维度筛选。
- 前端可查看和刷新股票宇宙。

建议数据表：

- `stock_universe`
- `stock_universe_sync_runs`

关键字段：

- `market`
- `symbol`
- `name`
- `exchange`
- `industry`
- `market_cap`
- `avg_turnover`
- `is_active`
- `is_blacklisted`
- `quality_seed_score`
- `source`
- `updated_at`

验收标准：

- 业务代码不再维护股票列表。
- 智能选股和模拟交易均从统一股票宇宙读取。
- 股票宇宙刷新失败时有可用兜底配置。

## 3.2 数据质量中心

目标：让行情、新闻、舆情、财报数据可追踪、可评分、可复盘。

功能：

- 对每条数据记录来源、采集时间、延迟、质量评分。
- 新闻/舆情去重，避免同一新闻多次影响情绪分。
- 对来源进行质量评分，例如官方公告 > 主流财经媒体 > 聚合源 > 社媒。
- 记录数据缺失、接口失败、模型失败。
- 前端展示数据源健康状态。

建议数据表：

- `data_events`
- `data_source_quality`
- `news_items`
- `market_quotes`
- `financial_snapshots`
- `social_sentiment_items`

验收标准：

- 任意一次选股结果可以追踪到当时使用的数据。
- 数据源失败不导致整个系统崩溃。
- 前端能看到当前数据是否可信。

当前落地：

- 已新增 `news_items`、`market_quote_snapshots`、`financial_snapshots`、`data_source_quality` 标准表。
- 新闻资讯会记录来源、类型、情绪分、相关性、新鲜度、来源权重和质量分。
- 行情总览会写入指数和股票行情快照。
- 策略评分会写入所用行情与财务快照。
- 已新增 `/api/data-quality/status`、`/api/data-quality/news`、`/api/data-quality/market-quotes`、`/api/data-quality/financials`。

待增强：

- 把每条 `factor_scores` 精确关联到新闻、行情、财务快照 ID。
- 增加失败数据源的错误上报和采集任务记录。
- 前端新增数据中心页面，展示数据覆盖率、来源质量和最近失败原因。

## 3.3 因子评分与策略引擎

目标：把选股逻辑从黑盒文本分析变成可解释、可调参的评分系统。

核心因子：

- 质量：ROE、毛利率、净利率、负债率、现金流。
- 成长：营收增速、利润增速、预期增速。
- 估值：PE、PB、PS、PEG、行业分位。
- 趋势：MA20、MA60、突破、成交量、波动率。
- 情绪：新闻情绪、社媒情绪、负面事件。
- 风险：回撤、波动、财务异常、舆情黑天鹅、流动性。

功能：

- 支持策略参数配置。
- 支持不同市场使用不同权重。
- 支持优质股榜和潜力股榜分开排序。
- 支持输出买入、观望、持有、卖出、规避。

验收标准：

- 每只候选股必须有因子拆解。
- 每个买入信号必须有入场价、止损价、止盈价、仓位比例。
- 低质量数据必须降低信号置信度。

当前落地：

- 已新增 `factor_scores` 表，保存候选股综合分、分项因子、交易计划、入选原因、风险提示和数据质量分。
- 自动交易运行会按 `run_id` 记录当轮候选评分，便于后续关联订单、持仓和复盘收益。
- 已新增 `/api/strategy/factors` 与 `/api/strategy/factors/latest` 查询接口。

待增强：

- 将评分记录关联到标准化行情、新闻、舆情、财务快照。
- 区分优质股榜和潜力股榜的权重、榜单类型和选股阈值。
- 增加因子有效性统计，用真实后验收益校准权重。

## 3.4 回测系统

目标：验证策略是否有机会接近年化 50%，并量化风险代价。

功能：

- 支持选择市场、时间区间、初始资金。
- 支持选择策略参数。
- 支持历史行情、财报、新闻情绪回放。
- 支持样本内训练、验证集调参、样本外测试，避免只看单次回测收益。
- 支持参数网格搜索，对比不同阈值、持仓数量、调仓周期、止损止盈和因子权重。
- 支持涨跌停、停牌、成交量不足、滑点、手续费、最低佣金等真实交易约束。
- 生成交易明细和净值曲线。
- 输出核心指标。

核心指标：

- 年化收益
- 最大回撤
- 夏普比率
- 信息比率
- 胜率
- 盈亏比
- 平均持仓周期
- 换手率
- 最大连续亏损次数
- 样本外年化收益
- 样本外最大回撤
- 样本外夏普
- 数据覆盖率

建议数据表：

- `backtest_runs`
- `backtest_trades`
- `backtest_equity_curve`
- `strategy_params`
- `strategy_experiments`
- `strategy_leaderboard`
- `strategy_param_trials`

验收标准：

- 可以完成至少 A 股策略的基础历史回测。
- 回测结果可保存、查询、对比。
- 前端能展示净值曲线和交易明细。

当前落地：

- 已新增 `backtest_runs`、`backtest_trades`、`backtest_equity_curve`。
- 已实现价格动量基线回测，按历史价格计算信号、调仓、净值、回撤和交易记录。
- 已新增 `/api/backtest/run`、`/api/backtest/runs`、`/api/backtest/runs/{run_id}`。
- 当前基线回测明确不使用当前财务和当前新闻，避免未来函数。

待增强：

- 接入历史财务和历史新闻/舆情快照后，再回放完整 AI 因子策略。
- 增加参数对比、基准指数对比和行业暴露统计。
- 增加样本内、验证集、样本外三段式验证。
- 增加真实交易约束：涨跌停、停牌、成交量、滑点、手续费、最低佣金。
- 增加策略排行榜，按样本外收益、回撤、夏普、胜率和稳定性综合排序。
- 前端新增回测复盘页面，展示净值曲线、回撤曲线、交易明细和失败原因。

## 3.4.1 Qlib 研究引擎接入

目标：复用 Qlib 的专业因子研究、实验记录和组合回测能力。本项目不重复造 Qlib 已经成熟的研究框架，而是负责数据整合、AI 解释、前端交互、复盘回填和新手可读的策略报告。

Qlib 负责：

- 因子数据集构建，例如 Alpha158、Alpha360 或自定义特征。
- 模型训练与预测，例如 LightGBM、线性模型、深度学习模型。
- Experiment/Recorder 实验记录。
- 组合回测和组合绩效分析。
- IC、RankIC、信息比率、最大回撤等专业指标。

本项目负责：

- 统一股票宇宙、新闻、舆情、财务、龙虎榜、模拟交易数据。
- 将本项目因子和 AI 情绪信号转成 Qlib 可用的信号或特征。
- 保存 Qlib 实验结果到业务数据库。
- 在前端展示策略排行榜、样本外表现、失败原因和 AI 解释。
- 将回测表现回填到训练样本和因子有效性统计。

验收标准：

- 可以使用 Qlib 对 A 股策略跑一次完整样本内/样本外实验。
- 同一策略的不同参数实验可对比。
- 前端策略排行榜不只展示收益，还展示回撤、夏普、胜率、换手率、样本外表现和数据覆盖率。
- Qlib 不可用时，本地基线回测仍能运行，但页面需要明确标记为“基础回测”。

## 3.5 模拟交易持久化增强

目标：服务重启后账户、持仓、订单、收益不丢失。

功能：

- 支持账户持久化。
- 支持订单持久化。
- 支持持仓快照。
- 支持每日净值记录。
- 支持多策略组合。
- 支持手动交易和自动交易统一入库。

建议数据表：

- `accounts`
- `positions`
- `orders`
- `portfolio_snapshots`
- `strategy_accounts`

验收标准：

- 服务重启后账户状态一致。
- 每次模拟交易后可查询订单和持仓。
- 盈亏、仓位、市值来自持久化数据，而不是内存。

## 3.6 复盘训练闭环

目标：让系统知道自己哪些判断有效，哪些判断失败。

功能：

- 每个候选股记录当时因子、新闻、情绪、价格。
- 自动回填 T+1、T+5、T+20、T+60 收益。
- 对错误信号分类。
- 统计因子有效性。
- 生成策略调参建议。

错误类型：

- 情绪误判
- 追高失败
- 趋势失效
- 财报质量不足
- 估值过高
- 止损过宽
- 仓位过重

建议数据表：

- `training_samples`
- `review_backfills`
- `factor_performance`
- `strategy_tuning_suggestions`

验收标准：

- 每次模拟运行都能生成训练样本。
- 复盘任务能自动回填收益。
- 前端能看到策略近期错误原因分布。

## 3.7 风险控制引擎

目标：避免系统为了追求高收益而忽视回撤和极端风险。

功能：

- 单仓上限。
- 总仓位上限。
- 行业集中度限制。
- 单日亏损熔断。
- 连续亏损暂停交易。
- 高波动降仓。
- 重大负面新闻强制降权。

验收标准：

- 任意买入动作必须通过风险检查。
- 风险拒绝买入时必须返回拒绝原因。
- 前端仪表板展示当前风险状态。

## 3.8 LLM 与 AI 分析降级

目标：避免模型余额不足或接口失败导致系统不可用。

功能：

- 支持模型供应商切换。
- 支持余额不足时自动降级到规则分析。
- 深度分析失败时返回结构化错误。
- 模型配置页显示当前模型、密钥状态、失败原因。
- 可配置不同任务使用不同模型。

验收标准：

- LLM 失败不影响行情、选股、模拟交易基本功能。
- 前端能明确显示 AI 分析是否使用了降级模式。

## 7. 非功能需求

- 页面首屏加载不超过 2 秒。
- 行情总览通过缓存和 WebSocket 降低等待。
- 所有外部数据源必须有超时和降级。
- 所有策略结果必须可审计。
- 关键配置不能写死在代码里。
- 不能因为 LLM 失败导致核心功能不可用。
- 涉及交易状态的写入必须具备幂等或事务保护。

## 8. 开发优先级

### P0 必做

- 账户、订单、持仓、收益持久化稳定。
- 股票宇宙数据库化。
- 智能选股因子拆解。
- 模拟交易训练样本持久化。
- 复盘回填稳定。
- 基础回测框架。

### P1 重要

- 数据质量中心。
- 前端回测复盘页。
- 风险控制引擎。
- LLM 降级机制。
- 策略参数配置页。

### P2 增强

- 多策略组合。
- 多账户模拟。
- 策略参数自动调优。
- 行业轮动分析。
- 社媒舆情扩展。

## 9. 第一阶段开发拆分

建议先做 5 个开发任务：

1. 股票宇宙数据库化
2. 账户与组合快照持久化
3. 因子评分结果标准化入库
4. 基础回测引擎
5. 回测复盘前端页面

第一阶段完成后，系统应能回答：

- 当前选股范围从哪里来？
- 为什么选中这 10 只股票？
- 模拟交易重启后是否还在？
- 这套策略历史表现如何？
- 哪些因子真正贡献了收益？

## 10. 验收标准

第二版完成时应满足：

- 服务重启后账户、持仓、订单、净值不丢失。
- 智能选股从统一股票宇宙读取。
- Top10 优质股和潜力股均有清晰解释。
- 回测结果可保存并在前端查看。
- 复盘任务能回填至少 T+1/T+5/T+20 收益。
- 数据质量状态可见。
- LLM 失败时系统仍可完成基础选股和模拟交易。
