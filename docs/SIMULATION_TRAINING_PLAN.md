# 模拟交易训练方案

## 1. 目标

本项目的模拟交易目标是尽量逼近年化 50%，但系统不会把 50% 当成固定承诺，而是把它作为训练和复盘的目标函数。

第一阶段先建立闭环：

```text
候选股票池
  -> 新闻/情绪/营收/估值/行情评分
  -> 模拟买入或观察
  -> 止损、止盈、情绪转弱卖出
  -> 写入复盘日志
  -> 根据实际表现调整权重和阈值
```

## 2. 当前策略

当前实现位于：

- `tradingAgents/trader/auto_strategy.py`
- `tradingAgents/server/routers/simulation.py`
- `frontend/src/pages/SimulationLab.tsx`

评分因子：

- 质量：ROE、毛利率、净利率、负债水平。
- 成长：营收增长、净利润增长。
- 估值：PE、PB。
- 行情：20 日收益、60 日收益、MA20、MA60、放量、突破。
- 情绪：个股新闻情绪均值、正负面新闻数量。
- 风险：高估值、高波动、负增长、负面新闻、跌破趋势。

默认权重：

```text
final_score =
  0.22 * quality
+ 0.18 * growth
+ 0.12 * valuation
+ 0.28 * momentum
+ 0.12 * sentiment
+ 0.08 * liquidity
- 0.18 * risk
```

买入条件：

- 综合分大于等于 72。
- 行情分大于等于 58。
- 情绪分大于等于 42。
- 风险等级不是 extreme。

卖出条件：

- 亏损达到 8%。
- 盈利达到 22%。
- 综合评分跌破持仓阈值。
- 新闻情绪明显转弱。

## 3. 前端工作台

新增页面：

```text
/simulation
```

页面包含：

- 年化 50% 目标进度。
- 当前候选股均分。
- 买入/观察信号数量。
- 止损止盈参数。
- 候选股评分列表。
- 单只股票评分拆解。
- 买入依据和风险提示。
- 新闻情绪样本。
- 最近模拟训练复盘记录。

## 4. 复盘存储

每次运行 `/api/simulation/run` 后，会追加写入：

```text
memory/simulation_runs.jsonl
```

每条记录包含：

- run_id。
- market。
- account。
- orders。
- decisions。
- top_candidates。
- review。

后续可在此基础上增加：

- 1 日、5 日、20 日实际收益。
- 相对指数超额收益。
- 最大回撤。
- 决策是否正确。
- 参数版本。
- 策略反思。

## 5. 下一阶段训练方式

下一步不是盲目提高买入频率，而是通过复盘调参：

1. 如果交易太少：
   - 降低买入阈值到 68。
   - 增加候选池。
   - 降低行情确认要求。

2. 如果收益不足但回撤可控：
   - 提高 momentum 和 event 权重。
   - 提高止盈目标。
   - 增加强势行业权重。

3. 如果回撤过大：
   - 提高买入阈值。
   - 降低单仓比例。
   - 收紧止损。
   - 增加负面新闻否决规则。

4. 如果经常错过大涨：
   - 增加突破和放量权重。
   - 引入行业热度。
   - 引入财报超预期事件。

## 6. 风险说明

年化 50% 是高目标，必须在模拟环境中通过长期回测和前向测试验证。系统当前用于模拟训练，不应用于真实自动下单。
