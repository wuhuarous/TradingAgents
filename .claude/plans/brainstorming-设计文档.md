# AI 量化交易系统 — 设计文档 v0.1

> 日期：2026-04-30
> 基于：用户需求 + TradingAgents（Apache 2.0）+ TradingAgents-CN（参考）

---

## 一、项目定位

AI 驱动的全自动量化交易系统，覆盖 A 股 + 港美股，使用免费行情数据，从数据采集→AI 分析→大V观点融合→自动交易→风控→复盘进化全链路自动化。初期使用模拟交易（真实行情+虚拟资金），验证策略有效性后再考虑实盘对接。

**核心差异化**：在 TradingAgents 多智能体分析基础上，增加大V经验融合 + AI 动态风控 + 策略自进化。

---

## 二、架构方案

**采用方案 B**：基于 TradingAgents（LangGraph 多智能体框架，Apache 2.0 许可证）作为分析核心引擎，自建数据层、交易层、调度层和前端。

```
┌────────────────────────────────────────────────────┐
│                    前端展示层                        │
│            (React/Vue SPA + 仪表盘)                 │
├────────────────────────────────────────────────────┤
│                 调度 & 自动化层                      │
│        (定时任务 / 事件驱动 / 任务队列)              │
├──────────┬──────────┬───────────┬─────────────────┤
│ 数据采集层│ AI 分析层 │ 交易执行层 │ 风控 & 复盘层   │
│          │          │           │                 │
│ A股行情   │ 多智能体 │ 策略引擎   │ AI动态风控       │
│ 港美股行情 │ 分析辩论  │ 模拟交易   │ 每日复盘总结     │
│ 财务数据   │ 大V融合   │ 订单管理   │ 策略参数自进化   │
│ 新闻舆情   │ 结构化输出│ 持仓跟踪   │ 经验记忆库       │
│ 大V观点   │          │           │                 │
├──────────┴──────────┴───────────┴─────────────────┤
│                    存储层                           │
│       SQLite (基础) + Redis (可选) + 文件缓存        │
├────────────────────────────────────────────────────┤
│               大模型供应商层（可切换）                │
│   DeepSeek / OpenAI / Claude / 兼容 OpenAI 接口     │
└────────────────────────────────────────────────────┘
```

## 四、核心流程

```
交易日 8:00
  │
  ├─ 数据采集（并行拉取行情/财务/新闻/大V）
  │     ├─ A股：AkShare/BaoStock
  │     ├─ 港美股：yfinance
  │     └─ 大V观点：爬虫+AI摘要
  │
  ├─ 全市场初筛（量价/技术/财务多维度）
  │
  ├─ 多智能体深度分析（LangGraph）
  │     ├─ 4位分析师 → 独立分析
  │     ├─ 多方/空方 辩论
  │     ├─ 风控三方 辩论
  │     ├─ 大V规则注入
  │     └─ 交易员综合决策
  │
  ├─ 9:00 前生成推荐报告（Top 5-10 + 买卖计划）
  │
9:15 集合竞价 / 9:30 开盘
  │
  ├─ 策略引擎生成交易计划
  ├─ 自动下单执行
  ├─ 盘中有变化 → AI 调整策略
  ├─ 风控触发 → 自动止损/止盈
  │
15:00 收盘
  │
  ├─ 当日结算报表
  ├─ AI 复盘 → 得失分析
  └─ 策略自进化 + 经验存入记忆库
```

## 五、技术选型

| 层 | 技术 | 理由 |
|-----|------|------|
| 分析引擎 | TradingAgents (LangGraph) | Apache 2.0，多智能体成熟框架 |
| 后端 | FastAPI | 异步支持好，与 TradingAgents-CN 参考一致 |
| 前端 | React + TypeScript | 生态丰富，适合数据密集型面板 |
| 数据库 | SQLite（主） + MongoDB（可选） | 个人工具从轻开始，SQLite 零部署 |
| 任务调度 | APScheduler + Celery（可选） | 轻量到重量可平滑升级 |
| 缓存 | Redis（可选） | 单机场景可先用文件缓存 |
| LLM | DeepSeek（默认）+ OpenAI + Claude | 多供应商可切换 |
| A股数据 | AkShare + BaoStock（备用Tushare） | 免费优先 |
| 港美股数据 | yfinance + Alpha Vantage | 免费优先 |
| 部署 | Docker Compose | 一键启动 |

## 八、设计决策记录

1. **为什么选 SQLite 而不是 MySQL/PostgreSQL？** — 个人工具，零运维。后期需要多用户时再迁移到 PG。
2. **为什么不用 TradingAgents-CN 的 app/ frontend/？** — 版权限制，其核心目录需商业授权。我们参考设计思路，自己实现。
3. **为什么保留 MongoDB 作为可选？** — TradingAgents 原版不用 MongoDB，但 A 股非结构化数据（新闻/公告）适合文档数据库。
4. **为什么大V模块用规则+爬取双路？** — 规则保证底线（不犯低级错误），爬取提供时效性（最新热点方向）。
